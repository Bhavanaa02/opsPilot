# Deployment Guide for OpsPilot\n\n## Overview\n\nThis guide covers deploying OpsPilot to production:\n- **Frontend:** Vercel (Recommended) or Netlify\n- **Backend:** Railway or Render\n- **Database:** Railway PostgreSQL or AWS RDS\n\n---\n\n## Prerequisites\n\n- GitHub account with opsPilot repository\n- Vercel account (vercel.com)\n- Railway account (railway.app) or Render account (render.com)\n- PostgreSQL database\n\n---\n\n## Part 1: Backend Deployment (Railway)\n\n### Step 1: Prepare Backend for Production\n\n1. **Create `backend/railway.json`:**\n```json\n{\n  \"$schema\": \"https://railway.app/railway.schema.json\",\n  \"build\": {\n    \"builder\": \"DOCKERFILE\"\n  },\n  \"deploy\": {\n    \"startCommand\": \"python -m uvicorn app.main:app --host 0.0.0.0 --port $PORT\"\n  }\n}\n```\n\n2. **Create `backend/Dockerfile`:**\n```dockerfile\nFROM python:3.11-slim\n\nWORKDIR /app\n\nCOPY requirements.txt .\nRUN pip install --no-cache-dir -r requirements.txt\n\nCOPY . .\n\nEXPOSE 8000\n\nCMD [\"python\", \"-m\", \"uvicorn\", \"app.main:app\", \"--host\", \"0.0.0.0\", \"--port\", \"8000\"]\n```\n\n3. **Create `backend/.dockerignore`:**\n```\n__pycache__\n*.pyc\n*.pyo\n*.pyd\n.Python\nvenv/\nenv/\n.env\n.env.local\n.git\n.gitignore\n```\n\n### Step 2: Deploy to Railway\n\n1. Go to [railway.app](https://railway.app)\n2. Click \"New Project\" → \"Deploy from GitHub\"\n3. Select your `opsPilot` repository\n4. Railway will detect `Dockerfile`\n5. Add environment variables:\n   - `DATABASE_URL`: PostgreSQL connection string\n   - `SECRET_KEY`: Generate a secure key\n   - `OPENAI_API_KEY`: (Optional) For LLM integration\n\n6. Click \"Deploy\"\n7. Your backend will be available at: `https://your-project.railway.app`\n\n### Step 3: Database Setup on Railway\n\n1. In Railway dashboard, click \"+ New\"\n2. Select \"Database\" → \"PostgreSQL\"\n3. Railway creates database automatically\n4. Copy the connection string to `DATABASE_URL`\n\n---\n\n## Part 2: Frontend Deployment (Vercel)\n\n### Step 1: Prepare Frontend\n\n1. **Update `frontend/.env.production`:**\n```env\nVITE_API_URL=https://your-backend.railway.app\n```\n\n2. **Update `frontend/src/services/api.ts`:**\n```typescript\nconst API_BASE_URL = process.env.VITE_API_URL || 'http://localhost:8000'\n```\n\n### Step 2: Deploy to Vercel\n\n1. Go to [vercel.com](https://vercel.com)\n2. Click \"New Project\"\n3. Import your GitHub repository\n4. Select `frontend` as root directory\n5. Add environment variables:\n   - `VITE_API_URL`: Your Railway backend URL\n6. Click \"Deploy\"\n7. Your frontend will be available at: `https://your-project.vercel.app`\n\n---\n\n## Part 3: Environment Variables\n\n### Backend (.env)\n```env\n# Database\nDATABASE_URL=postgresql://user:password@host:5432/opspilot\n\n# FastAPI\nAPP_NAME=OpsPilot\nAPP_VERSION=0.1.0\nDEBUG=False\n\n# Security\nSECRET_KEY=your-production-secret-key-here\nALGORITHM=HS256\nACCESS_TOKEN_EXPIRE_MINUTES=30\n\n# OpenAI (Optional)\nOPENAI_API_KEY=sk-your-api-key-here\n\n# CORS\nALLOWED_ORIGINS=https://your-frontend-domain.vercel.app,https://your-custom-domain.com\n```\n\n### Frontend (.env.production)\n```env\nVITE_API_URL=https://your-backend.railway.app\n```\n\n---\n\n## Part 4: Custom Domain Setup\n\n### Vercel (Frontend)\n\n1. Go to Project Settings → Domains\n2. Add your custom domain\n3. Follow Vercel's DNS setup instructions\n4. Update `ALLOWED_ORIGINS` in backend .env\n\n### Railway (Backend)\n\n1. Go to Deployments → Settings\n2. Add custom domain\n3. Configure DNS records\n\n---\n\n## Part 5: CI/CD Pipeline (GitHub Actions)\n\n### Create `.github/workflows/deploy.yml`:\n\n```yaml\nname: Deploy\n\non:\n  push:\n    branches:\n      - main\n\njobs:\n  deploy:\n    runs-on: ubuntu-latest\n    \n    steps:\n      - uses: actions/checkout@v3\n      \n      - name: Deploy Backend to Railway\n        run: |\n          npm i -g @railway/cli\n          railway up --service backend\n        env:\n          RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}\n      \n      - name: Deploy Frontend to Vercel\n        uses: vercel/action@master\n        with:\n          vercel-token: ${{ secrets.VERCEL_TOKEN }}\n          vercel-args: '--prod'\n          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}\n          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}\n          working-directory: ./frontend\n```\n\n---\n\n## Part 6: Monitoring & Maintenance\n\n### Backend Monitoring (Railway)\n\n- Dashboard shows:\n  - CPU & Memory usage\n  - Request logs\n  - Error tracking\n  - Uptime\n\n### Frontend Monitoring (Vercel)\n\n- Analytics:\n  - Page views\n  - Performance metrics\n  - Error tracking\n\n### Add Error Tracking (Sentry)\n\n1. Sign up at [sentry.io](https://sentry.io)\n2. Create new project\n3. Add to backend `requirements.txt`:\n   ```\n   sentry-sdk[fastapi]==1.38.0\n   ```\n\n4. Add to `backend/app/main.py`:\n   ```python\n   import sentry_sdk\n   from sentry_sdk.integrations.fastapi import FastApiIntegration\n   \n   sentry_sdk.init(\n       dsn=settings.SENTRY_DSN,\n       integrations=[FastApiIntegration()],\n       traces_sample_rate=1.0,\n   )\n   ```\n\n---\n\n## Part 7: Troubleshooting\n\n### Backend Won't Start\n\n```bash\n# Check logs\nrailway logs\n\n# Verify requirements.txt\npip install -r requirements.txt\n\n# Test locally\npython -m uvicorn app.main:app --reload\n```\n\n### Database Connection Issues\n\n```bash\n# Test connection string\npsql postgresql://user:password@host:5432/opspilot\n\n# Check DATABASE_URL in .env\necho $DATABASE_URL\n```\n\n### CORS Errors\n\n- Update `ALLOWED_ORIGINS` in backend .env\n- Restart backend service\n- Check frontend URL matches exactly\n\n---\n\n## Part 8: Performance Optimization\n\n### Backend\n\n```python\n# Add caching to database.py\nfrom sqlalchemy.orm import Session\nfrom functools import lru_cache\n\n@lru_cache(maxsize=128)\ndef get_cached_issues():\n    # Your cache logic\n    pass\n```\n\n### Frontend\n\n```bash\n# Optimize build\nnpm run build\n\n# Check bundle size\nnpm install -D bundlesize\n```\n\n---\n\n## URLs After Deployment\n\n- **Frontend:** `https://your-project.vercel.app`\n- **Backend API:** `https://your-project.railway.app`\n- **Swagger Docs:** `https://your-project.railway.app/docs`\n- **Health Check:** `https://your-project.railway.app/health`\n\n---\n\n## Next Steps\n\n- [ ] Set up GitHub Actions for CI/CD\n- [ ] Configure custom domains\n- [ ] Add error tracking (Sentry)\n- [ ] Set up monitoring alerts\n- [ ] Configure backups for database\n- [ ] Add SSL certificates\n\n---\n\n**Deployment Complete!** 🎉\n"
